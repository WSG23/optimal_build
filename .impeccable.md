# Optimal Build — Design Context

## Design Context

### Users
Commercial real estate professionals — agents sourcing and marketing properties, and developers running feasibility, acquisition, and finance workflows. Sophisticated, data-literate users who spend extended sessions making high-stakes decisions (site acquisition, financial modeling, compliance analysis). Used in office environments across Singapore, Hong Kong, New Zealand, Seattle, and Toronto. These users respect density and distrust tools that oversimplify.

### Brand Personality
**Sharp, assured, technical.**

The interface should feel like infrastructure you'd trust billions on. Engineering-grade precision with institutional gravitas. No hand-holding, no softness — this is a command surface for experts who already know what they're looking for.

**Emotional target:** Command and confidence. The user should feel total visibility and control, like sitting at a Bloomberg terminal where nothing is hidden and every data point is within reach.

### Aesthetic Direction
**Tone:** Mission-control precision. Dense, structured, and deliberate. Authority through information mastery, not decoration.

**Theme:** Dark and light, both first-class. Dark is the power-user hero experience; light must be equally refined, not an afterthought.

**References:**
- Bloomberg Terminal — information density, professional gravitas, keyboard-driven workflows, earned complexity
- The feeling of sitting in a trading floor war room with total situational awareness

**Anti-references:**
- Government/legacy apps — dense tables with no hierarchy, gray-on-gray, no visual rhythm, looks abandoned
- Generic SaaS dashboards with card grids, gradient accents, and hero metrics
- Flashy crypto/Web3 aesthetics with neon glows and glassmorphism overload
- Soft, rounded, hand-holding UIs (Salesforce/HubSpot style)

**Typography strategy:** Custom display font for headings and hero elements to elevate brand identity. System fonts for body text (performance + familiarity for data-heavy views).

**Existing design system:** Mature "Square Cyber-Minimalism" with Obsidian & Sapphire palette, 26 canonical components, comprehensive token system (`--ob-*` prefix), sharp border radii (2-4px default), glass morphism surfaces.

### Accessibility
WCAG 2.1 AA compliance — 4.5:1 contrast ratio for text, keyboard navigable, screen reader support. Dark and light modes must both meet contrast requirements independently.

### Design Principles

1. **Density over decoration.** Every pixel should carry information or create breathing room for information. No ornamental elements that don't serve comprehension. If a visual element doesn't help the user make a decision faster, remove it.

2. **Earned complexity.** Show the full picture to users who want it. Progressive disclosure is for onboarding, not for hiding capability. Power users should never feel the tool is holding back.

3. **Structured authority.** Clear visual hierarchy through typography weight, spatial rhythm, and color restraint — not through decorative borders, gradients, or shadows. The data IS the design.

4. **Institutional trust.** The interface should feel like it was built by engineers for engineers. No trends, no flash. Timeless precision that would look at home on an institutional trading desk or in a civil engineering control room.

5. **Dual-mode parity.** Dark and light themes are equal citizens. Neither is a degraded version of the other. Both must feel intentional and complete.
